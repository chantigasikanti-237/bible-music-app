Home Screen
Dashboard that greets the user, shows their last reading position (via history API), a verse of the day, and quick-access tiles for the main sections.
1
Load last position — calls GET /api/history/last on mount. Renders Continue Reading card with a LinearProgressIndicator showing chapter progress.
2
Verse of the Day — pulled from GET /api/scripture/chapter. Displayed in the forest-green accent card.
3
Quick Access tiles — Bible, Music, Bookmarks. Tapping navigates to the relevant tab via AppShellController.
API calls on this screen
GET/api/history/last🔒 auth
GET/api/scripture/chapterVerse of day
Bible Reader
Chapter-by-chapter reading with verse highlighting, prev/next chapter navigation, and long-press to bookmark a verse. Whole-book audio downloads available.
1
Fetch chapter — GET /api/v1/bibles/:versionId/books/:bookId/chapters/:chNum. Shows clayLoadingCenter() while loading.
2
Save history — on chapter open, calls POST /api/history/update to persist reading position.
3
Long-press verse — shows success snackbar "Verse saved" after POST /api/verses/save.
API calls on this screen
GET/api/v1/bibles/:id/books/:book/chapters/:ch
POST/api/history/update🔒 auth
POST/api/verses/save🔒 auth
Music / Worship
Language-filtered song list with YouTube audio streaming proxy. Mini player persists across tabs. Mutual exclusion with Bible audio player.
1
Load songs — GET /api/audio/songs/:language. Filter chips switch language. Shows clayLoadingCenter() on first load.
2
Stream audio — tapping a song calls GET /api/audio/url/:videoId then streams via proxy.
3
Search — GET /api/audio/search?q= debounced at 400ms.
API calls on this screen
GET/api/audio/songs/:language
GET/api/audio/search?q=
GET/api/audio/url/:videoId
GET/api/audio/stream/:videoIdProxy stream
Profile
User stats, bookmarked verses management, reading streak, and sign-out. All data requires JWT auth.
1
Load bookmarks — GET /api/v1/users/me/bookmarks. Swipe-to-delete triggers DELETE /api/verses/:id.
2
Load history — GET /api/v1/users/me/history for reading streak and stats.
3
Sign out — calls POST /api/auth/logout then clears local session.
API calls on this screen
GET/api/v1/users/me/bookmarks🔒 auth
DEL/api/v1/users/me/bookmarks/:id🔒 auth
GET/api/v1/users/me/history🔒 auth
POST/api/auth/logout🔒 auth
Auth — Login / Register
Clay-styled input fields, primary forest-green CTA, forgot-password flow. Rate-limited endpoints with CSRF protection on state-mutating calls.
1
Login — POST /api/auth/login returns access + refresh tokens. Stored in secure HttpOnly cookies.
2
Register — POST /api/auth/register with name, email, password. Shows success snackbar on completion.
3
Token refresh — POST /api/auth/refresh fires automatically when access token expires (401 interceptor in ApiClient).
API calls on this screen
POST/api/auth/loginrate limited
POST/api/auth/registerrate limited
POST/api/auth/refreshCSRF protected
POST/api/auth/password-reset/request
[ 04 — API ]

API Endpoint Reference

Node.js + Express backend running on port 5000. MongoDB + Redis. 🔒 = JWT required. ⚡ = rate limited.

Authentication — /api/auth
MethodEndpointFlagsDescription
POST/api/auth/login⚡ rateLogin, returns tokens
POST/api/auth/register⚡ rateCreate account
POST/api/auth/refreshCSRFRefresh access token
POST/api/auth/logout🔒 authInvalidate session
POST/api/auth/password-reset/request⚡ rateSend reset email
POST/api/auth/password-reset/confirm⚡ rateSet new password
Users — /api/v1/users
MethodEndpointFlagsDescription
GET/api/v1/users/me🔒 authCurrent user profile
GET/api/v1/users/me/bookmarks🔒 authList bookmarks
POST/api/v1/users/me/bookmarks🔒 authCreate bookmark
DEL/api/v1/users/me/bookmarks/:id🔒 authDelete bookmark
GET/api/v1/users/me/history🔒 authReading history
POST/api/v1/users/me/history🔒 authAdd history entry
